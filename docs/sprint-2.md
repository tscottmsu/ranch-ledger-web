# Sprint 2: Ranch Setup

## Goal

Allow the Ranch Administrator to build the core setup data needed before daily operations can begin.

## Included

- Employees
- Horses
- Trails
- Activity Types
- Setup progress on the administrator dashboard
- Left sidebar navigation for the Ranch Administrator dashboard

## Not Included

- Guest management
- Reservations
- Today's Operations
- Activity scheduling
- Horse/guest assignment engine
- Maps
- Live GPS
- Emergency beacon

## Acceptance Criteria

- Ranch Administrator can add, edit, and archive employees.
- Ranch Administrator can add, edit, and archive horses.
- Ranch Administrator can add, edit, and archive trails.
- Ranch Administrator can add, edit, and archive activity types.
- Dashboard shows setup progress.
- All records are scoped to the current ranch.
- Protected routes remain protected.
- RLS is respected.
- TypeScript, ESLint, and production build pass.
- Dashboard uses a left sidebar navigation layout.
- Sidebar includes Dashboard, Employees, Horses, Trails, Activity Types, and Settings.
- Sidebar is responsive and usable on desktop and mobile.
