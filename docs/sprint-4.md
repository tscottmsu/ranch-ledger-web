# Sprint 4: Daily Operations Foundation

## Goal

Build the first operational workflow for Ranch Ledger and continue improving the product’s brand identity.

## Included

- Activities CRUD
- Today’s Operations page
- Basic activity scheduling
- Activity status tracking
- Sidebar navigation updates
- Homepage/marketing page improvement
- More realistic operational preview cards
- Warmer, less generic Ranch Ledger branding

## Activity Fields

- Activity Type
- Date
- Start Time
- End Time
- Trail
- Capacity
- Status
- Notes

## Activity Statuses

- Draft
- Ready
- In Progress
- Completed
- Cancelled

## Today’s Operations

The page should help the Head Wrangler quickly understand:

- What activities are scheduled today
- Which activities are still drafts
- Which are ready
- Which are active
- Which are completed
- Which guests are checked in or ready for operations

## Brand Direction

Ranch Ledger should feel like premium outdoor operations software.

Not generic SaaS.
Not cartoon western.
Not cold admin template.

Use:

- Rust orange
- Charcoal
- Warm ivory
- Soft white surfaces
- Green only as a status color, not the main brand color

Homepage should feel more balanced and less left-heavy.

The hero should tell the story of a ranch morning and show a realistic operational preview.

## Not Included

- Assignment engine
- Guest-to-activity assignment
- Horse assignment
- Wrangler assignment
- Maps
- Live GPS
- Emergency beacon
- Reports

## Acceptance Criteria

- Ranch Administrator can create/edit/archive activities.
- Head Wrangler can view and manage today’s activities.
- Activities are scoped to the current ranch.
- Today’s Operations page exists.
- Sidebar includes Today’s Operations and Activities.
- Homepage feels warmer, more branded, and less generic.
- TypeScript, ESLint, and production build pass.
